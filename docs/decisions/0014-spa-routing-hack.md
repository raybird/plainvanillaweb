# ADR 0014: GitHub Pages SPA 路由修復 (SPA 404 Hack)

## 上下文
GitHub Pages 作為靜態檔案託管服務，並不支援伺服器端路由 (Server-side Routing)。這導致單頁應用 (SPA) 在使用非根目錄路徑（如 `/search` 而非 `/#/search`）時，伺服器會因找不到對應的靜態檔案而回傳 404 錯誤。

## 決策
利用 GitHub Pages 的自訂 404 頁面機制進行客戶端路由修復。
1.  **自訂 404.html**：當伺服器回傳 404 時，此頁面會被載入。
2.  **JS 自動導向**：腳本會解析當前的 `pathname`，並將其轉換為帶有 `#/` 的 Hash 路由格式，然後重定向回 `index.html`。

## 後果
- **優點**: 解決了直接訪問子路徑或重新整理頁面導致的 404 問題，提升使用者體驗。
- **優點**: 無需配置複雜的伺服器重寫規則 (Rewrite Rules)，完全依賴靜態文件。
- **缺點**: 首次載入可能會有短暫的跳轉延遲。
- **缺點**: 需要硬編碼 Repository 名稱 (如 `/plainvanillaweb`) 以正確解析路徑。
