# ADR 0017: 原生輕量級國際化系統 (Native Lightweight i18n System)

## 上下文
為了讓專案具備多語言支援能力，同時保持「零依賴」與「輕量化」的設計哲學。我們需要一個簡單但功能完備的國際化解決方案。

## 決策
1.  **I18nService**: 實作一個單例服務 `lib/i18n-service.js`。
    *   負責非同步載入 JSON 語言檔 (`assets/locales/*.json`)。
    *   管理當前語言狀態並持久化至 `localStorage`。
    *   提供 `$t(key)` 翻譯函式，支援巢狀屬性存取。
2.  **BaseComponent 整合**:
    *   修改 `BaseComponent`，使其自動訂閱 `i18n` 的 `change` 事件。
    *   當語言變更時，自動觸發 `update()` 重新渲染組件。
3.  **資源分離**: 翻譯內容與程式碼分離，存放於靜態 JSON 檔案中。

## 後果
- **優點**: 極致輕量，無需引入大型第三方函式庫。
- **優點**: 自動化的響應式翻譯，開發者只需使用 `$t()` 即可。
- **缺點**: 每次切換語言會觸發全頁面重繪 (Full Repaint)，對於超大型應用可能有效能影響（但在本專案規模可忽略）。
