# ADR 0033: 原生 Canvas 高效能圖表系統 (Native High-Performance Charting)

## 上下文
應用程式需要即時展示性能指標（如記憶體、Web Vitals 趨勢）。在無框架開發中，若使用 DOM 元素或 SVG 進行高頻率更新，會導致顯著的重繪開銷。

## 決策
實作基於 **Canvas API** 的即時圖表渲染系統：
1.  **CanvasChart 類別**: 封裝 2D Context 的繪圖邏輯，實現一個輕量級的時間序列圖表渲染器。
2.  **效能優化**: 圖表更新時僅清除 Canvas 畫布並重新繪製像素路徑，而非操作 DOM 樹，大幅減少 CPU 與記憶體壓力。
3.  **零依賴整合**: 展示如何僅用少量原始碼實作關鍵業務功能，無需引入 D3.js 或 Chart.js 等重型庫。
4.  **教學頁面**: 建立 `AnalyticsPage` 展示技術實踐，並對比 Canvas 與 DOM 的性能差異。

## 後果
- **優點**: 提供流暢的 60FPS 即時數據更新，體現極致效能。
- **優點**: 應用的啟動體積幾乎不受影響。
- **優點**: 具備高度自訂性，可精確控制每一個像素的呈現。
- **缺點**: Canvas 內容是非語義化的 (Non-semantic)，需額外處理無障礙 (A11y) 標籤（如為 Canvas 提供 `aria-label` 或替代文字）。
