# ADR 0037: 原生虛擬列表組件 (Native Virtual List Component)

## 上下文
在處理大量數據（如 GitHub 搜尋結果，可能達上百筆）時，一次性渲染所有 DOM 節點會導致顯著的性能下降，增加記憶體消耗並造成捲動卡頓。

## 決策
實作 `<v-list>` (Virtual List) 組件：
1.  **視窗化渲染 (Windowing)**: 僅渲染當前捲動容器可見區域內的項目。
2.  **動態偏移**: 使用一個絕對定位的隱藏元素撐開總高度，並透過 `translateY` 動態移動項目容器到正確位置。
3.  **反應式更新**: 整合 `BaseComponent` 的反應式狀態，當捲動位置改變時自動計算並重新渲染可見項目。
4.  **零框架實作**: 完全不依賴第三方庫，僅使用原生 DOM 事件與 CSS。

## 後果
- **優點**: 極大提升了大數據量下的渲染效能與捲動流暢度。
- **優點**: 顯著減少 DOM 節點數量，優化記憶體佔用。
- **優點**: 展現了在 Vanilla 環境下實作複雜 UI 組件的能力。
- **缺點**: 需要固定每個項目的高度（Item Height），對於變高項目的處理較為複雜。
