# ADR 0046: File System Access API 本地開發環境整合

## 上下文
目前專案的 Playground 雖能即時執行程式碼，但資料儲存在暫存中，無法直接持久化至使用者的硬碟。傳統方案是透過 `<input type="file">` 上傳或 `<a>` 下載，但這種交互方式對於開發流程極不流暢。

## 決策
1.  **原生目錄存取**: 採用 `window.showDirectoryPicker()`，讓使用者能授權 Playground 存取本機專案目錄。
2.  **直接儲存 (Save-in-place)**: 實作 `FileSystemFileHandle` 的 `createWritable` 串流寫入，讓開發者在瀏覽器中按下 Ctrl+S 即可直接修改硬碟檔案。
3.  **檔案監控**: 建立簡單的檔案變更輪詢或事件監控機制，實現與本地編輯器同步。

## 後果
- **優點**: 徹底模糊了網頁應用與原生軟體的邊界，將 Playground 提升為一個具備「生產力工具」屬性的開發環境。
- **優點**: 完美契合 Vanilla 的「無工具鏈」開發哲學，讓瀏覽器成為唯一的開發工具。
- **缺點**: 受限於瀏覽器安全策略，必須在 HTTPS 下且經使用者明確互動授權。
