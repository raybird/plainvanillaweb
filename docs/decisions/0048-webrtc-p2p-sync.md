# ADR 0048: WebRTC P2P 無伺服器通訊架構

## 上下文
大多數 Web 應用依賴中心化伺服器進行數據交換。但在「原生 Web」開發中，瀏覽器其實具備建立直接「對等連線 (Peer-to-Peer)」的能力。為了展示無伺服器架構的極限，我們需要探索一種 P2P 通訊方案。

## 決策
1.  **採用 WebRTC 資料通道 (DataChannel)**: 直接利用 `RTCPeerConnection` 建立 P2P 資料傳輸。
2.  **純前端信令 (Signaling) 範例**: 提供一個可由手動複製/貼上 SDP (Session Description Protocol) 的互動介面，展示「零伺服器」連線建立過程。
3.  **無中心化檔案傳輸**: 在實驗室中實作一個「區域網快傳」工具，將數據直接由一台裝置的瀏覽器發送到另一台，不經過任何雲端中介。

## 狀態
**Accepted (2026-02-18)**

## 實作備註
已建立 `webrtc-service.js` 並於實驗室實作「手動信令聊天室」範例。

## 後果
- **優點**: 展示了瀏覽器在實作「隱私保護」與「去中心化」應用上的技術廣度。
- **優點**: 實現了完全零後端開銷的高速即時數據同步。
- **缺點**: WebRTC 的連線建立 (ICE/STUN/TURN) 在複雜網路環境（如對稱型 NAT）中可能失敗，需額外的中轉伺服器。
- **缺點**: SDP 手動複製/貼上對普通使用者來說操作過於複雜。
