# ADR 0053: 響應式設計與行動體驗優化策略

## 上下文
隨著 `plainvanillaweb` 功能日益豐富（如 Playground、Lab），其介面複雜度也隨之增加。目前部分版面（如雙欄編輯器、Grid 列表）在手機直立螢幕上顯得擁擠且難以操作。為了符合現代 Web 標準，我們必須確保應用在任何尺寸下都具備卓越的可用性。

## 決策
1.  **流式佈局優先 (Fluid Layout)**: 採用 CSS Grid `minmax` 與 Flexbox `wrap` 特性，盡量減少對固定媒體查詢 (Media Queries) 的依賴。
2.  **觸控友善 (Touch Friendly)**: 所有互動按鈕與連結的點擊熱區至少為 44x44px，並增加元件間距以防誤觸。
3.  **單欄堆疊策略**: 在小於 768px 的裝置上，強制將並排內容（如 Playground 的代碼/預覽、Docs 的導覽/內容）轉為垂直堆疊。
4.  **字體與可讀性**: 針對小螢幕調大基礎字級 (16px+) 並增加行高，確保長時間閱讀的舒適度。

## 後果
- **優點**: 顯著提升行動用戶的使用者體驗與留存率。
- **優點**: 符合 Google Core Web Vitals 與 SEO 的行動裝置友善標準。
- **缺點**: 需要對現有所有頁面組件進行 CSS 重構與測試。
