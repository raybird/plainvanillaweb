# ADR 0053: 原生 Payment Request API 整合

## 上下文
在電子商務網站中，結帳流程往往是流失率最高的環節。傳統的表單填寫繁瑣且體驗不一。W3C 推出的 `Payment Request API` 旨在提供一個標準化、瀏覽器原生的支付介面，讓使用者能快速選取已儲存的地址與信用卡資訊進行結帳。

## 決策
1.  **實作 PaymentService**: 封裝 `PaymentRequest` 建構子與 `show()` 方法，提供建立支付單據、處理運送選項與收集支付憑證的統一接口。
2.  **模擬支付流程**: 由於真實支付需要後端處理，本專案將實作一個「模擬結帳」範例，展示從建立訂單到獲取 `paymentResponse` 的完整前端流程。
3.  **實驗室展示**: 在 Lab 中提供「原生結帳」範例，允許使用者自訂商品項目並體驗瀏覽器彈出的原生支付視窗。

## 後果
- **優點**: 極大簡化了前端支付表單的開發工作。
- **優點**: 提供一致且安全的原生 UI，提升使用者信任感與結帳轉化率。
- **缺點**: 實際扣款仍需配合 Stripe、PayPal 等支付閘道的 SDK 或後端 API。
- **缺點**: 瀏覽器支援度與 UI 呈現方式在不同平台（如 Chrome vs Safari）可能存在差異。
