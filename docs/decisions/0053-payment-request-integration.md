# ADR 0053: 原生 Payment Request API 整合

## 上下文
在電子商務場景中，結帳流程的繁瑣往往是導致使用者流失的主因。傳統的表單輸入不僅體驗不佳，且難以與系統級支付方式（如 Apple Pay, Google Pay）無縫整合。現代瀏覽器提供的 Payment Request API 旨在標準化這一流程，提供一致且快速的支付體驗。

## 決策
1.  **實作 PaymentService**: 封裝 `PaymentRequest` 建構子與 `show()` 方法，提供統一的支付請求介面。
2.  **支援多種支付方式**: 預設支援 `basic-card`（基本信用卡），並保留擴展至第三方支付 App 的彈性。
3.  **教學單元實作**: 在實驗室 (Lab) 中提供「模擬結帳」範例，展示如何定義商品項目、運費選項與處理支付回應。

## 後果
- **優點**: 顯著提升行動裝置上的結帳體驗，利用系統儲存的資訊快速完成支付。
- **優點**: 減少開發者自行維護複雜表單驗證的需求。
- **缺點**: `basic-card` 在某些新版瀏覽器（如 Chrome）中已棄用，需轉向依賴 Google Pay 等具體支付方法，或僅作為教學展示。
- **缺點**: 需要 HTTPS 環境才能運作。
