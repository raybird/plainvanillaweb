# ADR 0054: 響應式設計與行動體驗優化策略

## 上下文
隨著 `plainvanillaweb` 的功能日益豐富，其在行動裝置上的閱讀與操作體驗逐漸暴露出短板。為了確保專案能適應各種螢幕尺寸，並提供一致且優質的使用者體驗 (UX)，我們需要一套明確的響應式設計策略。

## 決策
1.  **Mobile-First 策略**: CSS 開發優先針對小螢幕設計，再透過 `@media (min-width: ...)` 逐步增強大螢幕佈局。
2.  **統一斷點規範**: 
    - `768px`: 手機/平板分界，主要用於切換單欄/多欄佈局與導覽列模式。
    - `900px`: 導覽列完全展開的閾值。
3.  **觸控友善**: 
    - 所有互動元素 (按鈕、連結) 最小高度需為 `44px`。
    - 增加元素間距，避免誤觸。
4.  **彈性佈局**: 全面採用 Flexbox 與 Grid，並使用 `gap` 屬性管理間距，取代固定的 `margin`。

## 後果
- **優點**: 顯著提升行動裝置用戶的滿意度與留存率。
- **優點**: 代碼結構更清晰，減少了針對特定裝置的 hack。
- **缺點**: 需要對現有的 CSS 進行大規模重構（已完成）。
- **缺點**: 在複雜組件（如 WebRTC 視訊流）的佈局上需花費更多心思處理比例問題。
