# ADR 0055: 原生螢幕錄製與串流整合

## 上下文
在遠端協作與教學場景中，螢幕分享與錄製是核心需求。傳統方案往往依賴瀏覽器擴充功能或專用軟體。然而，現代 Web API (Screen Capture API, MediaStream Recording API) 已具備強大的原生能力，允許網頁直接擷取、預覽並錄製使用者的螢幕畫面。

## 決策
1.  **實作 MediaService**: 封裝 `navigator.mediaDevices.getDisplayMedia` 與 `MediaRecorder`，提供統一的螢幕擷取與錄製接口。
2.  **串流處理**: 支援將擷取到的 `MediaStream` 直接導向 `<video>` 元素進行即時預覽。
3.  **錄製與下載**: 實作將串流數據收集為 `Blob` 並觸發下載的完整流程。
4.  **教學單元實作**: 在實驗室 (Lab) 中提供「螢幕錄影機」範例，展示從權限請求到影片下載的全過程。

## 後果
- **優點**: 實現了零安裝、跨平台的螢幕錄製工具，大幅提升了 Web 應用的實用性。
- **優點**: 展示了瀏覽器在處理高頻寬多媒體數據流上的原生能力。
- **缺點**: `getDisplayMedia` 必須由使用者手勢觸發，且在行動裝置上的支援度有限（通常僅支援相機而非螢幕分享）。
- **缺點**: 錄製長影片可能佔用大量記憶體（需實作分塊處理或提醒）。
