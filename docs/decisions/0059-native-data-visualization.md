# ADR 0059: 實作原生 SVG 圖表引擎

## 狀態
已提議 (Proposed)

## 背景
數據可視化是工業級應用的標準配置。目前的解決方案通常涉及引入數百 KB 的第三方庫（如 Chart.js, Highcharts）。在 `plainvanillaweb` 中，我們希望證明僅使用原生 SVG 與數學計算，即可實現具備高效能與互動性的圖表。

## 決策
實作一個基於 SVG 的輕量級圖表引擎。
1. **技術選擇**：使用 SVG 而非 Canvas，以便利用 CSS Transitions 處理數據更新動畫，並維持良好的可存取性 (A11y)。
2. **路徑算法**：實作線性比例尺 (Linear Scaling) 算法，將原始數據轉換為 SVG 座標。
3. **反應式整合**：圖表組件將與 `BaseComponent` 的反應式狀態綁定，當 `data` 陣列改變時自動重繪路徑。

## 後果
- **優點**：極致的加載速度 (零依賴)、完全可自定義的樣式、與 CSS 動畫完美整合。
- **缺點**：對於極其複雜的圖表類型（如 3D 或 力導向圖），開發難度較高。
