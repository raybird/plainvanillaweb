# ADR 0062: 實作原生響應式表單引擎

## 狀態
已提議 (Proposed)

## 背景
表單處理與驗證是 Web 應用的核心痛點。雖然瀏覽器提供 Constraint Validation API，但在處理複雜邏輯（如：密碼二次確認、伺服器端唯一性檢查、欄位互動狀態）時仍力有未逮。開發者往往因此轉向 React Hook Form 或 Angular Reactive Forms 等庫。

## 決策
實作一個專業級的原生響應式表單引擎。
1. **核心模型**：仿照工業標準，將表單分為 `FormControl` (單一欄位) 與 `FormGroup` (欄位組合)。
2. **狀態追蹤**：自動追蹤 `valid` (合法), `invalid` (非法), `touched` (已觸碰), `dirty` (已修改), `pending` (驗證中)。
3. **驗證機制**：支援同步 (Sync) 與非同步 (Async) 驗證器。支援事件驅動的實時回饋。
4. **設計哲學**：不操作 DOM，僅處理數據狀態與規則邏輯，透過事件與 UI 組件連動。

## 後果
- **優點**：提供可預測的表單行為，大幅減少處理輸入邏輯的 boilerplate 代碼，提升 Vanilla 應用的工程化水平。
- **缺點**：相較於簡單的 HTML 屬性驗證，學習曲線略高。
