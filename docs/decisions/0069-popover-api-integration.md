# ADR 0069: 引入原生 Popover API 互動層

## 狀態

Accepted

## 背景

專案已具備多種原生 Web API 範例，但在「輕量互動層」方面仍主要依賴傳統自訂彈窗邏輯。為了延續 Vanilla-first 原則，需要一個更貼近平台能力、又能保持可維護性的彈出層方案。

## 決策

採用瀏覽器原生 `Popover API` 作為 Lab 的新示範項目，並同時示範兩種實作模式：

1. **宣告式模式**：使用 `popovertarget` 與 `popover` 屬性完成基本觸發與關閉。
2. **程式式模式**：使用 `showPopover()` / `hidePopover()` 將彈出層接入既有狀態流程。
3. **降級策略**：在不支援環境提供明確提示，不阻斷頁面其他功能。

## 後果

- **優點**：減少客製彈窗程式碼，提升互動一致性與可維護性。
- **優點**：能直接教學「平台原生互動 API」的宣告式與程式式整合方式。
- **缺點**：舊版瀏覽器支援度有限，仍需保留降級顯示策略。
