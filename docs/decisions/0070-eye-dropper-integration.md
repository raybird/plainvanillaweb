# ADR 0070: 引入原生 EyeDropper API 取色體驗

## 狀態

Accepted

## 背景

專案已完成多項原生 API 教學，但缺少「設計工作流」面向的能力示範。為了讓 Vanilla-first 教學覆蓋 UI 主題與視覺配置場景，需要加入可直接從畫面取色的原生 API。

## 決策

導入 `EyeDropper API` 作為 Lab 新項目，並以最小依賴方式整合：

1. 在 `#/lab/eyedropper` 提供互動式取色流程。
2. 取色後更新本地色票與預覽卡片，展示 API 與狀態管理整合。
3. 在不支援環境顯示降級提示，維持頁面可用性。

## 後果

- **優點**：擴充教學維度，涵蓋設計 token 與主題色管理場景。
- **優點**：維持零第三方依賴，符合 Vanilla 原始精神。
- **缺點**：瀏覽器支援度仍有限，需明確標示相容性與替代方案。
