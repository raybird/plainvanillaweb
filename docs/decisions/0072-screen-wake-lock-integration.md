# ADR 0072: 引入原生 Screen Wake Lock API 保持喚醒

## 狀態

Accepted

## 背景

專案已有媒體擷取與掃碼等行動端教學場景，常遇到螢幕自動熄滅中斷流程。為了提升長流程操作的穩定性與教學完整度，需要導入瀏覽器原生的保持喚醒能力。

## 決策

導入 `Screen Wake Lock API` 作為 Lab 新項目，採用下列策略：

1. 在 `#/lab/wakelock` 提供手動啟用與釋放操作。
2. 監聽 `visibilitychange`，頁面回前景時可選擇自動重新申請。
3. 顯示持續時間與最後釋放原因，幫助理解系統釋放行為。
4. 不支援環境提供明確降級提示，維持頁面可用性。

## 後果

- **優點**：改善長時間展示與掃碼教學體驗，降低自動熄屏中斷。
- **優點**：維持零第三方依賴，符合 Vanilla-first。
- **缺點**：API 支援度與系統省電策略差異大，需以提示與回復策略補足。
