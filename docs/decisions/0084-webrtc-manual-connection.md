# ADR 0084: WebRTC 手動連線流程優化

## 狀態
已接受 (Accepted)

## 背景
在沒有信令伺服器 (Signaling Server) 的情況下，WebRTC P2P 連線需要使用者手動交換 SDP。原先的 `WebRTCPage.js` 在 SDP 顯示與操作流程上不夠直觀，導致實際測試難度較高。

## 決策
重構 `WebRTCPage.js` 的交互介面與邏輯。
1. **一鍵複製**：新增 `copyLocalSDP()` 方法，利用 `navigator.clipboard` 讓使用者快速獲取本地產出的 SDP。
2. **對應回應**：新增 `createRTCAnswer()` 方法，支援「收到 Offer 後產出對應 Answer」的標準流程。
3. **介面分層**：將連線控制區與對話互動區分離，並加入步驟指引。
4. **狀態連動**：確保連線成功後 UI 自動轉為對話模式。

## 後果
- **優點**：大幅降低 WebRTC 原生實驗的測試門檻，讓開發者在不建置伺服器的情況下也能驗證 P2P 通訊能力。
- **缺點**：手動交換 SDP 依然受限於網路環境（如對稱型 NAT），但在實驗室情境下已具備極高的實作價值。
