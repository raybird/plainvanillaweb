# ADR 0095: 原生 Service 交互模式與注入規範 (Standardized Patterns)

## 狀態
已接受 (Accepted)

## 背景
專案已進入「教育與架構」階段。雖然組件已具備深度反應式能力，但組件與 Service 之間的通訊模式（如事件訂閱、狀態同步）尚未在實驗室中進行直觀展示與教學化。

## 決策
建立一個專屬的「Service 交互模式」實驗室頁面：
1. **Singleton 注入示範**：展示組件如何顯式匯入 Service 單例並與其互動。
2. **事件流視覺化**：實作一個通訊軌跡儀表板，即時呈現 Service 發布事件與組件接收更新的過程。
3. **取消訂閱自動化**：示範利用 `BaseComponent` 的生命週期與 `BaseService.on` 返回的閉包進行資源清理。
4. **三位一體同步**：建立 `ServicePatternPage.js`，撰寫 `service-patterns.md` 實踐篇。

## 後果
- **優點**：為開發者提供了一套可預測、易於偵錯且零耦合的原生開發範式。
- **缺點**：過度依賴事件流可能導致數據追蹤變得複雜，需強調單向數據流原則。
