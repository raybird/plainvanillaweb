# ADR 0096: 原生 Vanilla 宣言與長青代碼準則實作

## 狀態
已接受 (Accepted)

## 背景
專案已進入「教育與生態」的收尾階段。雖然技術實作已極其豐富，但缺乏一個核心思想的總結，即「為什麼要選擇 Vanilla？」。為了將專案提升至方法論層次，需要正式實作 Vanilla Manifesto 頁面與相關教育文檔。

## 決策
實作 Vanilla Manifesto 與教育引導系統：
1. **Manifesto 頁面**：建立 `ManifestoPage.js`，以極簡主義美學呈現「長青代碼」的核心價值觀。
2. **長青代碼準則**：撰寫 `MANIFESTO.md`，定義零依賴、標準優先、最小抽象等原則。
3. **收尾同步**：將宣言正式掛載至主導覽與 ROADMAP，完成教育階段的最後拼圖。

## 後果
- **優點**：為專案注入靈魂，從「工具集」升級為「開發思想體系」。
- **缺點**：純理論內容較多，需透過優質的 UI 排版維持吸引力。
