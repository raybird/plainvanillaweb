# 國際化系統 (i18n)

為了保持輕量，我們沒有使用任何第三方 i18n 函式庫，而是實作了一個原生的 `I18nService`。

## 架構設計

1.  **資源分離**: 翻譯檔為標準 JSON 格式，存放於 `assets/locales/` (如 `zh-TW.json`, `en-US.json`)。
2.  **非同步載入**: 僅在需要時透過 `fetch` 載入語言檔，並快取於記憶體中。
3.  **狀態同步**: 語言偏好儲存於 `localStorage`，重新整理後保持不變。

## 使用方式

所有繼承自 `BaseComponent` 的組件都自動獲得 `$t` 方法與響應式更新能力。

```javascript
// 在組件中
render() {
    return html`
        <button>${this.$t('app.submit')}</button>
    `;
}
```

當語言切換 (`i18n.setLocale('en-US')`) 時，`BaseComponent` 會自動觸發 `update()`，重新渲染 UI 以顯示新的語言內容。
